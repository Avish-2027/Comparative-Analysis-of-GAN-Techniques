# Comparative Analysis of GAN Techniques for Face Generation

This project presents a comparative analysis and implementation of three different Generative Adversarial Network (GAN) architectures for human face generation:

- DCGAN — Deep Convolutional Generative Adversarial Network
- ACGAN — Auxiliary Classifier Generative Adversarial Network
- BEGAN — Boundary Equilibrium Generative Adversarial Network

The objective of this project is to understand and compare different GAN architectures by implementing them using PyTorch and analyzing their generated outputs.

---

## Project Overview

Generative Adversarial Networks (GANs) are deep learning models designed to generate new data samples that resemble real-world data. A GAN generally consists of two neural networks:

1. Generator
2. Discriminator

The Generator learns to create realistic synthetic images from random noise, while the Discriminator learns to distinguish between real images from the dataset and fake images generated by the Generator.

In this project, three different GAN architectures are implemented and studied:

### 1. DCGAN

Deep Convolutional GAN (DCGAN) improves the original GAN architecture by using convolutional neural networks for image generation and discrimination.

The Generator uses transposed convolutional layers to generate images from random latent vectors, while the Discriminator uses convolutional layers to classify images as real or fake.

Implementation:

`train_dcgan.py`

---

### 2. ACGAN

Auxiliary Classifier GAN (ACGAN) extends the traditional GAN architecture by incorporating class information into the generation process.

The Generator receives both random noise and class labels to generate class-conditioned images. The Discriminator performs two tasks:

- Determines whether an image is real or fake.
- Predicts the class of the input image.

This approach provides additional control over the generated images.

Implementation:

`train_acgan.py`

---

### 3. BEGAN

Boundary Equilibrium GAN (BEGAN) introduces an autoencoder-based discriminator and an equilibrium mechanism to balance the training process between the Generator and Discriminator.

Instead of directly classifying images as real or fake, the discriminator evaluates reconstruction losses. The equilibrium mechanism helps maintain a balance between the Generator and Discriminator during training.

Implementation:

`train_began.py`

---

## Dataset

The models are trained using the UTKFace dataset.

The UTKFace dataset contains a large collection of face images along with demographic information such as age, gender, and ethnicity.

For this project, the face images are used to train the GAN models to learn the distribution of human facial features and generate synthetic face images.

Before running the models, download the UTKFace dataset and update the dataset path in the respective Python files if required.

## Model Comparison

The three implemented architectures differ in their approach to image generation and training.

| Feature | DCGAN | ACGAN | BEGAN |
|---|---|---|---|
| Convolutional Architecture | Yes | Yes | Yes |
| Random Noise Input | Yes | Yes | Yes |
| Class-Conditioned Generation | No | Yes | No |
| Auxiliary Classification | No | Yes | No |
| Autoencoder-Based Discriminator | No | No | Yes |
| Equilibrium-Based Training | No | No | Yes |

### DCGAN

DCGAN provides a strong convolutional baseline for image generation. It learns to generate images using a Generator and Discriminator trained in an adversarial manner.

### ACGAN

ACGAN extends the GAN framework by introducing class labels. This enables the Generator to produce images based on specific class information and gives the Discriminator an additional classification task.

### BEGAN

BEGAN uses an autoencoder-based discriminator and an equilibrium mechanism to balance the training process. This provides a different approach to improving the training behavior of GANs.

## Experimental Setup

The models are implemented and trained using PyTorch.

The experiments use image-based training with the following general configuration:

- Image Size: 64 × 64
- Batch Size: 64
- Latent Vector Size: 100
- Learning Rate: 0.0002
- Number of Epochs: 5
- Framework: PyTorch

The exact configuration may vary between the individual GAN implementations depending on the architecture and training requirements.

## Comparative Analysis

The main goal of this project is to study how different GAN architectures approach the problem of synthetic face generation.

### DCGAN

DCGAN focuses on using convolutional architectures to improve the quality of generated images compared to traditional fully connected GAN architectures.

### ACGAN

ACGAN introduces class conditioning, allowing the model to incorporate additional information during image generation. This provides more control over the generated outputs.

### BEGAN

BEGAN introduces an autoencoder-based discriminator and an equilibrium-based training mechanism. This provides an alternative approach to balancing the Generator and Discriminator during training.

Overall, the project demonstrates that different GAN architectures use different strategies to address challenges in generative modeling, including image quality, controllability, and training dynamics.
